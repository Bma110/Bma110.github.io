下载安装

略

工作原理

从数据库导出文献题录；【需要结合你找文献的行为】

文献题录分为TXT文件和非TXT的endnote文件（.RIS.NBIB.CIW etc.）；【找到那些文献后寻找导出的botton】

endnote识别导出的文件并且存入你的库（endnote library）；【在endnote中选择导入你之前导出的文件】

输出期刊要求的文献格式；

形成参考文献。

导出文献

PubMed：Citation Manager导出

在PubMed导出200份文献以内使用citation manager。

你的search result界面有send to选项；

点击send to并选择其下citation manager和create file；

Create 后生产的那个文件你打开就自动导入到你endnote库页面。（建议建好库再去导）

EndNote导入文献

过滤器（filter）

在导入不同数据库导出的TXT时，需要使用不同的filter，使EN正确读取其结构而正确地导入文献。

不用filter的格式：nbib、rie、ciw

在官网你可以下载更多的filter，放到安装目录下的filters文件目录下即可使用。

具体步骤

File-import-file-import option-选择filter，如果没有你要的请选择otherfilters继续选择。

手动创建题录

适用于无法导出的文献（网页、书本章节）

References-new references-reference type-web page还是啥-依次往下填：作者、发表日期、题目……

field的自定义

在编辑-首选项里面改

比如我加了个是否需要deep read，EN会自动关联显示reference里面的相关tab内容。

在field上右键你会看到更多东西，这将有助于你快速获得文献关键信息和管理。

建库

没建库时界面什么也没有。

点击文件-新建，在弹框选择储存路径和重命名你的库（enl为后缀名“EndnoteLibrary”），点击保存完成建库。

变化：

在你选择的位置出现了一个enl后缀的文件（类似一个清单）和一个同名文件夹（放文献原件）。

软件界面出现行的窗口。

文献管理

分组

endnote分组有两个级别：

  • 按组别：项目一、项目二……
  • 按类别：背景、方法、数据……

排序

按照field排序，只需要点击一下field。

排序顺序为作者、年份、收录号码，如果再点击相关field，则对应field排序等级提前。

如果具有唯一性，则会只根据此排序。

在工具里面你可以自定义你的排序。

添加全文附件

常规操作

选择需要下载全文的文献

点查找

你会看到几个状态：

  • 查找ing
  • 找到PDF
  • 找到URL
  • 未找到

未找到文献的解决办法：

  • 添加doi再次尝试
  • 手动下载然后点击

进行添加，或者直接拖进来。

批量导入文献

对已经有PDF的文献进行整理，特别是首次使用或者对未归类的PDF文献。

方法一

编辑-首选项-PDF处理-启用自动导入（以后文件夹更新重启库，内容自动导入，导入效果取决于内容和联网）

方法二

文件-导入-选择文件或者文件夹

EN会自动匹配。

题目更新（find reference update）

添加DOI后

可以根据DOI更新文献的相关信息比如URL

通过PDF导入的同样可以通过题目更新完善信息

如果你的DOI错了那么找的文献也会错

自动查找全文失败？这里还有一招

在导入文献题录之后，下一步就是查找全文，然后在开始阅读。

不过在我的使用过程中，中文文献在endnote中的使用体验实在不好，特别是知网导进去的文献，不过这个问题我已经有其他解决方法了。

对于英文文献，利用endnote会有一些无法自动查找到的全文，这时候就需要发挥共享精神了。如果搜不到的文献是高分的sci，那么这个方法可以解决你的问题，当然其他文献亦可一试。

相信不少科研工作者都经常使用sci-hub，这个网站的文章竟然是根据文献的DOI进行收录的。利用这个特点，我们在endnote中再自定义一个preview，就可以直接访问要sci-hub的原文了。

▼

按图示路径打开

第二步，新建样式，选择参考文献模板输入网址: “sci-hub.tf/插入字段DOI ”。

最后保存退出，打开文献的preview你会发现惊喜。
